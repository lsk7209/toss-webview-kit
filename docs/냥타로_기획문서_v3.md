## 0. v3 개요: 무엇이 달라졌나

v2는 "무엇을 만들지"는 잘 재단했지만 **"어떻게 만들고 어떻게 검증할지"**가 약했습니다. 2차 3인 팀의 크리틱으로 v3에서 다음이 추가·수정되었습니다.

| 범주 | v2 상태 | v3 개선 |
|------|---------|---------|
| **개발 일정** | 3주 MVP (비현실) | **12주 MVP** (일러스트 병렬) |
| **캐릭터 바이블** | "별도 세션 필요" | **22마리 전체 표 + 4대 세력 그룹** 포함 |
| **브랜딩** | 개별 파스텔 고양이 22마리 | **마스코트 지정 + 세력별 시각 차별화** |
| **Hook Model** | 4요소 언급 | **내부 트리거 5종 + 초기 3일 투자감 가속** |
| **재방문** | 시간대 3회 + 스트릭 | + **감정 트리거 설계 + 마이크로 인터랙션 8~12초** |
| **KPI** | 중위값 목표 | **상위 25% 지향 + A/B 테스트 로드맵** |
| **그로스** | 토스 홈 광고 집중 | **유입 채널 매트릭스 5종 + 전환 퍼널 설계** |
| **공유 카드** | 1200×600 Canvas | **세로형 SNS 포맷 추가 + 디자인 가이드** |
| **시즌 IP** | 없음 | **할로윈·크리스마스 로드맵** |
| **수익 모델** | 광고 시뮬레이션 | + **유저당 LTV 계산** |
| **네이밍** | "냥타로" 고정 | **검색 경쟁 검토 + 보조 네이밍** |

## 1. 2차 3인 팀 구성

1차 팀(김디렉터·박수익·이리텐)이 **"무엇을 얼마나 만들지"**를 정했다면, 2차 팀은 **"어떻게 만들고 어떻게 성공시킬지"**에 집중합니다.

| 역할 | 담당 | 핵심 책임 |
|------|------|----------|
| **박지연** | UX 리서처·행동 심리 | Hook Model 검증, 인지 부담 관리, 마이크로 인터랙션 |
| **조현우** | 크리에이티브 디렉터·브랜딩 | 캐릭터 바이블, IP 확장성, 시각 시스템, 마스코트 |
| **최다은** | 그로스·데이터 전략 | 가설 기반 KPI, 전환 퍼널, A/B 테스트, LTV, 유입 매트릭스 |

## 2. 2차 3인 크리틱 세션

### 2.1 박지연 크리틱 (UX·행동 심리)

**🔴 v2의 빈 구석**

1. **시간대별 3회 훅의 현실성 붕괴**
   - 유저가 토스 앱을 여는 주 목적: 송금·결제·포인트 확인. 냥타로를 "열 이유"가 충분히 설계 안 됨.
   - "저녁에 문득 냥타로 생각이 난다"는 **내부 트리거(internal trigger)**가 없음. 현재 v2의 트리거는 모두 외부 (배지·푸시).
   - 냥 버그 (Nir Eyal 프레임): _앱이 정착하려면 유저가 **감정적 허전함 → 앱 연상** 고리가 걸려야 한다._ 이게 없음.

2. **카드 뽑기 연출 시간 명시 안 됨**
   - 너무 빠름(<5초) → 의미 희석, 눌러도 "허탈"
   - 너무 김(>15초) → 이탈, 다음 날 재진입 꺼림
   - 습관형 앱 검증 구간: **8~12초** (뽑기 시작 → 카드 공개 → 메시지 읽기)

3. **컬렉션 그리드 22슬롯의 심리학적 문제**
   - 한 화면에 22장 전부 표시 = "한눈에 진척도가 눈에 박히지만 빨리 지침"
   - **Pacing**: 달성마다 "진척감 스파이크"가 생기게 설계해야 함. 화면 스크롤 1~2번 필요한 그리드가 더 효과적.
   - "실루엣 + 이름만 가려짐"은 약함. **실루엣 + 한 줄 힌트**로 FOMO 강화 필요.

4. **초기 3일 유저의 투자감 공백**
   - 스트릭·컬렉션 모두 "쌓이는 데 시간이 드는" 메커니즘. 초기 유저는 **쌓이기 전에 이탈**.
   - 첫 3일 이내에 "이미 뭔가를 잃을 수도 있다"는 감각을 심어야 함.

**🟢 박지연 제안**

1. **내부 트리거 5종 설계**
   - 😔 **허전함** → "오늘 한마디가 필요해" (저녁 외로움)
   - 😰 **불안** → "오늘 뭐 해야 할지 모르겠을 때" (아침 모호함)
   - 🤔 **호기심** → "내가 안 만난 냥이 뭐 있지?" (컬렉션 미완)
   - 💬 **공유욕** → "친구한테 웃긴 결과 보여주고 싶다" (재미있는 카드가 떴을 때)
   - 😎 **수집욕** → "오늘 이 시간대 냥이는 처음 보네?" (새 냥이 알림)

   각각에 대응하는 **앱 내 문구·시각 요소**를 상시 배치.

2. **카드 뽑기 연출 8~12초 세분화**
   ```
   0.0~1.5초  : 유저가 카드 더미 탭 → 햅틱 피드백
   1.5~3.5초  : 카드 3장이 원형으로 흩어짐 → 흐름 애니메이션
   3.5~5.0초  : 유저가 1장 선택 (또는 자동 선택)
   5.0~8.0초  : 카드 뒤집기 연출 + 냥이 실루엣 등장 → 파스텔 이펙트
   8.0~10초   : 냥이 이름 공개 + 시그니처 대사 1줄
   10~12초    : 메시지 본문 페이드인 + 공유·홈 버튼 등장
   ```
   - 저사양 기기 대응: 3.5초 이후 단계를 즉시 스킵 가능 ("바로 보기" 버튼)
   - 햅틱·사운드 off 시에도 같은 리듬 유지

3. **컬렉션 UX 재설계**
   - 22슬롯 → **3그룹 × 7~8슬롯** (세력별 탭). 세력 전환으로 스크롤 감 있음.
   - 미만난 슬롯: 실루엣 + **시그니처 대사 1줄** 살짝 보이기 (힌트). 예: "무너져도 돼..."만 보이면 유저는 "이 냥이 뭐지" 궁금증 유발.
   - 만난 횟수 + 가장 최근 만난 날짜 표시.

4. **초기 3일 투자감 가속 (온보딩 부스트)**
   - Day 1: 첫 뽑기 직후 "첫 번째 냥이 친구가 됐어요" 모멘텀. 컬렉션 1/22 즉시 표시.
   - Day 2: 앱 재진입 시 "어제 만난 냥이가 기다리고 있어요" (어제 냥이 상세로 유도)
   - Day 3: "스트릭 3일째! 이번 주 반 왔어요" 모멘텀 + 공유 유도 프롬프트
   - 7일 완주 시 **첫 주간 메달** (영구 뱃지).

### 2.2 조현우 크리틱 (크리에이티브·브랜딩)

**🔴 v2의 빈 구석**

1. **일러스트 제작 일정 비현실적**
   - 22장 고퀄리티 일러스트는 **최소 6~8주** (캐릭터 디자이너 1명 기준, 주 3~4장). v2의 "W1~2에 다 제작" 암시는 불가능.
   - "3주 MVP" 일정은 **일러스트 없이 출시**하거나 **저퀄 외주**를 전제로 하는데 둘 다 첫인상 파괴.

2. **시각적 차별화 부재**
   - "파스텔 고양이 22마리"는 결국 **비슷비슷하게 보임**. 색만 다른 고양이는 수집욕을 못 자극.
   - 타로 카드 상징이 냥이 뒤로 숨어버리면 "왜 이게 XVI 탑인지" 전달 안 됨.

3. **마스코트 부재**
   - 앱 아이콘 / 스플래시 / 온보딩에 쓸 **"얼굴마담" 1마리**가 없음.
   - 마스코트 없으면 PR·광고 크리에이티브 매번 다른 캐릭터로 찍어야 함 = 인지도 누적 불가.

4. **시즌 IP·확장성 빠짐**
   - 고양이 IP는 시즌 확장이 수익 구조의 핵심. 할로윈 냥이, 크리스마스 냥이, 설날 냥이, 벚꽃 냥이... v2에서 이 얘기가 없음.
   - IAP의 진짜 엔진은 "한정판 시즌 냥이". 프리미엄 팩 ₩2,900에 11마리 + 그 안에 시즌 한정 = IAP 구매율 상승.

5. **공유 카드 포맷이 단수**
   - v2는 1200×600 가로형만 언급. SNS 실태는 인스타 스토리(9:16), 카카오톡 공유(정사각), 트위터(16:9) 3중 포맷.
   - 가로형만 내면 인스타 공유가 떨어짐 (20~34세 여성 핵심 타겟에서 인스타 공유는 바이럴 1순위).

6. **네이밍 경쟁 미검토**
   - "냥타로"로 구글·앱스토어 검색 시 동명 또는 유사 앱 다수. 토스 내 노출 최적화에도 불리.

**🟢 조현우 제안**

1. **현실적 일러스트 일정 (6~8주 병렬)**
   - **W1~2**: 캐릭터 바이블 확정 (22마리 이름·성격·컬러·실루엣 차별화 포인트)
   - **W3~10**: 일러스트 제작 (주 3장 속도, 마스코트 1장은 최우선 주 1)
   - **개발 병렬 진행**: W1~8 동안 개발은 플레이스홀더(실루엣 임시)로 진행. W9~10에 실제 에셋 교체.
   - **v2.0 풀덱 78장**: 메이저 22 이후에도 주 3장 페이스로 **19주 추가**. 현실 반영해 v2.0 "+8주"는 "+22주"로 수정 (아래 일정표 참조).

2. **4대 세력 그룹 (시각 차별화)**
   - 타로의 4원소 (불·물·공기·흙)를 고양이 세력으로 번안
   - 각 세력별 **컬러 팔레트·배경 테마·성격 톤** 고정

   | 세력 | 컬러 | 성격 | 해당 메이저 아르카나 |
   |------|------|------|---------------------|
   | 🔥 **불냥이단** (Fire Pack) | 따뜻한 오렌지·코랄 | 활기, 돌진, 추진 | 황제, 전차, 힘, 태양, 심판 |
   | 💧 **물냥이단** (Water Pack) | 차분한 블루·민트 | 공감, 치유, 직관 | 여사제, 여황제, 달, 별, 매달린 자 |
   | 🌬️ **바람냥이단** (Air Pack) | 청명한 라벤더·그레이 | 지혜, 소통, 위트 | 마법사, 연인, 정의, 세계, 바보 |
   | 🌱 **땅냥이단** (Earth Pack) | 포근한 베이지·모스 | 안정, 현실, 물질 | 교황, 은둔자, 운명의 수레바퀴, 절제, 죽음 |

   > 탑(The Tower)과 악마(The Devil)는 **무소속 아웃사이더** 2마리로 분류 (세력 비뚤기 = 캐릭터성 강화).
   > 총 20 + 무소속 2 = 22 매핑 완료.

3. **공식 마스코트: "참치" (The Fool = 0번 카드)**
   - 호기심 가득한 코숏 (검정+흰색). 목에 빨간 반다나.
   - 앱 아이콘, 스플래시, 온보딩 화면 고정 등장.
   - 마케팅 이미지의 얼굴마담.
   - "참치의 카드 리딩 방송"이라는 식의 가상 컨셉으로 확장성 부여.

4. **시즌 IP 로드맵**

   | 시즌 | 시기 | 한정 냥이 | IAP |
   |------|------|----------|-----|
   | 🎃 할로윈 | 10월 중순 ~ 11월 초 | 5마리 (호박·마녀·뱀파이어 등) | 할로윈 팩 ₩2,900 |
   | 🎄 크리스마스 | 12월 중순 ~ 1월 초 | 5마리 (산타·루돌프·엘프 등) | 크리스마스 팩 ₩2,900 |
   | 🐉 설날 | 2월 초 | 3마리 (떡국·세배·복주머니) | 무료 배포 (리텐션용) |
   | 🌸 벚꽃 | 3월 말 ~ 4월 중 | 5마리 | 벚꽃 팩 ₩1,900 |
   | ☀️ 여름휴가 | 7월 말 ~ 8월 중 | 5마리 (수박·해변·수영) | 여름 팩 ₩2,900 |
   | ⭐️ 결정적 순간 | 연말 연초 | 시즌 총결산 12마리 세트 | 연말 팩 ₩3,900 |

   - 시즌 냥이는 **영구 컬렉션**에 포함 (미구매자는 공개 안 됨 — 이는 가챠가 아님, 확정 구매).
   - **무료 배포 시즌** (설날) = 리텐션 자극용 + 충성도 형성.

5. **공유 카드 3중 포맷**

   | 포맷 | 용도 | 크기 |
   |------|------|------|
   | 가로 배너형 | OG 이미지 (토스 공유) | 1200×600 |
   | **세로 스토리형** | **인스타 스토리** | 1080×1920 |
   | 정사각형 | 카카오톡·일반 공유 | 1080×1080 |

   - 셋 모두 Canvas로 실시간 생성. 3-tier fallback (Canvas → SVG → 정적 템플릿).
   - 공유 카드에 반드시 포함: 냥이 일러스트 / 카드 이름 / 짧은 한 줄 메시지 / 앱 로고+배지 / "토스에서 냥타로 검색"

6. **네이밍 재검토**
   - **"냥타로"**: 검색 경쟁 치열 (동명 웹툰·SNS 계정 다수). 토스 내 유니크 검색성 약함.
   - **대안 1**: "냥냥타로" (한정적 차별)
   - **대안 2**: "오늘의 냥이" (직관적, 시간대 훅과 결합)
   - **대안 3**: "참치의 타로 가게" (마스코트 기반 스토리텔링, 가장 IP 친화적)
   - **권장**: 정식명 "오늘의 냥이" / 부제 "참치의 타로 가게" / appName: `today-cat` 또는 `chamchitarot`

### 2.3 최다은 크리틱 (그로스·데이터)

**🔴 v2의 빈 구석**

1. **KPI가 "평균값"**
   - D1 35% / D7 12% / D30 4%는 카지노·소셜 앱 벤치마크 **중위값**. 습관형 엔터 앱 상위 25% 목표치는 더 공격적 (D7 18~20% / D30 7~10%).

2. **전환 지표 미설정**
   - 앱인토스 콘솔의 "전환 지표" 기능(핵심 1개 + 보조 2개) 활용 계획 없음.
   - 핵심 전환이 뭔지 정해야 추천·노출 최적화 신호가 생김.

3. **A/B 테스트 로드맵 부재**

5. **LTV 계산 없음**
   - 유저당 평생 가치를 모르면 마케팅 예산 의사결정 불가. CAC 대비 LTV 비율(LTV:CAC)이 최소 3:1 돼야 건강한 경제성.

6. **v2.0 바이럴 이월의 기회비용 미고려**

### 3.4 Hook Model 재강화

```

  내부: 허전함 / 불안 / 호기심 / 공유욕 / 수집욕 (앞에서 정의)
    ↓
Action
  탭 한 번으로 카드 뽑기 (마찰 최소)
    ↓
Variable Reward
  3종 가변성
    - 어떤 냥이가 나올지 모름 (22장 랜덤)
    - 메시지가 오늘 내 상황에 맞을지 (해석 가변성)
    - 레어 냥이 해금 조건 충족했을지 (v1.1~, 진척도 기반)
    ↓
Investment
  컬렉션 채워나감 / 스트릭 쌓아감 / 단짝 냥이 진화 (v1.1~)
    ↓ 다시 Trigger로 (시간대 바뀌면)
```

### 3.6 화면 구조 (v1.0, 인지 부담 최소화)

```
/intro              # 첫 진입 1회만. 참치 등장 + 앱 소개 3슬라이드
/home               # 홈 (메인)
/draw               # 카드 뽑기 연출 화면
```

총 **8개 화면** (v2 대비 변동 없음). 단, **홈 화면 정보 밀도 대폭 축소**:
- 상단: 참치 로고 + 오늘 날짜
- 중앙: 현재 시간대 카드 (빈 상태면 "뽑기" CTA, 뽑은 상태면 결과 요약)

- 배너 광고: 하단 요약 아래

초기 진입 유저가 **3초 내에 "뭘 하는 앱인지" 이해 가능** 수준으로 단순화.

### 3.7 컬렉션 UX (v3 개선)

#### 세력별 3탭 구조
- 탭 1: **불+물** (10마리 — 뜨거운·차가운)
- 탭 2: **바람+땅** (10마리 — 경쾌한·묵직한)
- 탭 3: **아웃사이더** (2마리)

탭별 7~10마리, 스크롤 1~2회 필요. 한 화면 다 보이지 않음 → **수집 과정 감각 유지**.

#### 슬롯 상태 3종

```
[만남]     컬러 일러스트 + 이름 + "N번 만남" + 마지막 날짜
[힌트]     실루엣 + "○○의 한 마디..." (시그니처 대사 앞 3~5글자만)
[미발견]   회색 실루엣 + "??? "  (진척도 3마리 이상 일 때만 힌트 해금)
```
